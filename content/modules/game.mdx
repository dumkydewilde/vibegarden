## What it is

A small game that runs in the browser: a quiz, a word puzzle, a memory game, a tiny arcade thing for your kids. Games are a brilliant first build because everything about them is visible: if it works, you see it, and if it is broken, you see that too. No accounts, no data, no deployment headaches required.

## When to use it

- You want to experience the build loop (describe, run, adjust) without worrying about real data.
- You have an audience: kids, a quiz night, a birthday, the club's holiday party.
- You want a personal twist on a known game: a quiz about your family, a wordle with your friend group's inside jokes.

## Setup steps

1. Pick a game you can explain in two sentences. "Cards face down, find the pairs, count the tries." If it takes a paragraph, it is too big for a first version.
2. Ask an AI assistant for it as one single HTML file. One file means you can save it, double-click it, and it runs.
3. Play it immediately, then change one thing at a time: colors, difficulty, your own questions or images.
4. Share it: send the file, or put it on a free static host so it has a link.

## Options and costs

Free. A single-file browser game costs nothing to run or host. You pay only for the AI calls while building, which for a small game is pennies. The skill you are actually practicing, describing behavior precisely and iterating in small steps, is exactly the skill every bigger project needs.
