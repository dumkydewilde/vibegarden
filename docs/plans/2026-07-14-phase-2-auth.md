# Vibe Garden Phase 2: Auth, Invites, Admin

**Goal:** Invite-only login with email OTP, sessions, and a working admin panel backed by D1.

**Decisions (header):**
- D1 via Drizzle ORM; migrations generated by drizzle-kit into `drizzle/`, applied with `wrangler d1 migrations apply`.
- Session: `vg_session` HTTP-only cookie holding `<sessionId>.<hmac>`; HMAC-SHA256 with `SESSION_SECRET` (WebCrypto). Sessions table, 30 days.
- OTP: 6 digits, 10 min expiry, max 5 attempts, one active code per email (resend replaces). Allowed for invited emails and `ADMIN_EMAIL` (bootstrap, becomes admin role).
- Mail: `sendOtpEmail` uses Resend when `RESEND_API_KEY` is set; otherwise the login action returns the code and the UI shows it with a "mail not configured" note. Good for local dev and pre-DNS production.
- Google OAuth: standard code flow at `/auth/google` + `/auth/google/callback`, only offered when `GOOGLE_CLIENT_ID`/`GOOGLE_CLIENT_SECRET` are configured. Google users still need an invite (or admin email).
- Access: all app-layout routes require login (redirect `/login`); `/login` and `/join` stay public. Admin routes require role admin. Invite status flips to `joined` on first login.

## Tasks

1. **D1 + Drizzle**: wrangler.jsonc `DB` binding (+ `ADMIN_EMAIL` var, `migrations_dir`), `app/db/schema.ts` (users, invites, sessions, otp_codes), `drizzle.config.ts`, npm scripts `db:generate` / `db:migrate` / `db:migrate:prod`, generated initial migration, local apply. `workers/app.ts` passes `{ cloudflare: { env, ctx } }` load context.
2. **Auth core**: `app/lib/auth.server.ts` (sign/verify cookie value, create/destroy session, getUser, requireUser, requireAdmin), `app/lib/otp.server.ts` (requestCode, verifyCode, upsert user), `app/lib/mailer.server.ts`. Vitest for sign/verify and OTP validation logic.
3. **Routes**: `/login` (email step, code step, dev-code display), `/logout` action, Google OAuth pair, app-layout loader guard + user in context, nav user chip with logout, admin-only nav filtering.
4. **Admin**: loader with participants (users joined with invite status) + pending invites; actions: add invite, revoke invite. Browser-verified end to end (invite -> login as invitee -> appears as joined).
